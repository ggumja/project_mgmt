# Jeisys B2B 쇼핑몰 프로젝트관리 Plan Document

> Version: 1.0.0 | Created: 2026-02-11 | Status: Draft

## 1. Executive Summary
B2B 쇼핑몰 개발 프로젝트의 생산성과 정확성을 높이기 위해, 기능정의서(FS)와 요구사항정의서(SRS)를 디지털화하여 관리하고 문서 간의 연관관계를 추적할 수 있는 웹 기반 관리 시스템을 구축합니다.

## 2. Goals and Objectives
- **문서 통합 관리**: 파편화된 Excel/Word 문서를 통합 데이터베이스로 관리
- **연관관계 추적(Traceability)**: 요구사항이 어떤 기능으로 구현되었는지 실시간 매핑 확인
- **워크플로우 자동화**: 승인 프로세스 및 버전 관리를 시스템화하여 휴먼 에러 방지
- **B2B 특화**: B2B 쇼핑몰만의 복잡한 가격 정책, 회원 등급, 견적 시스템 등의 템플릿 기본 제공

## 3. Scope
### In Scope
- **문서 CRUD**: 기능정의서 및 요구사항정의서 등록, 수정, 삭제
- **버전 관리**: 변경 이력(History) 저장 및 이전 버전 비교
- **연관관계 매핑**: 요구사항 ID와 기능 ID 간의 다대다(N:M) 연결
- **대시보드**: 프로젝트 진행률(구현율, 승인율) 시각화
- **승인 시스템**: 담당자 작성 → 검토자 승인 워크플로우

### Out of Scope
- 실시간 동시 편집 (Google Docs 방식) - Phase 3 고려
- 외부 도구(Jira/Confluence)와의 상용 연동 API 개발 - Phase 2 고려

## 4. Success Criteria
| Criterion | Metric | Target |
|-----------|--------|--------|
| 문서 작성 효율 | 수기 작성 대비 소요 시간 | 30% 감소 |
| 데이터 정합성 | 요구사항-기능 누락률 | 0% (시스템 강제 매핑) |
| 사용자 만족도 | 내부 개발팀 만족도 점수 | 4.5 / 5.0 |

## 5. Timeline (Phase 1: MVP)
| Milestone | Date | Description |
|-----------|------|-------------|
| 프로젝트 셋업 | Week 1 | Supabase 설정 및 프론트엔드 기본 구조 생성 |
| 데이터 스키마 | Week 2 | 문서, 버전, 연관관계 DB 설계 및 RLS 적용 |
| 핵심 기능 개발 | Week 3-5 | CRUD, 매핑 기능, 승인 워크플로우 구현 |
| QA 및 배포 | Week 6 | 통합 테스트 및 베타 서버 배포 |

## 6. Risks
| Risk | Impact | Mitigation |
|------|--------|------------|
| 데이터 복잡도 | 높음 | 정교한 Supabase 스키마 설계 및 타입 자동 생성 활용 |
| 사용자 초기 진입 장벽 | 중간 | 기존 템플릿(Markdown 기반) UI를 최대한 친숙하게 구현 |
| 요구사항 변경 | 중간 | 유연한 버전 관리 및 변경 이력 추적 기능 강화 |

## 💡 bkit 추가 제언
1. **AI 기반 품질 체크**: 작성된 요구사항이 모호하거나 기능 정의와 상충될 경우 AI가 자동으로 경고를 주는 기능을 추가하면 프로젝트 품질을 획기적으로 높일 수 있습니다.

3. **기존 문서 마이그레이션**: 현재 `docs/b2b_feature_list.md`에 있는 방대한 데이터를 시스템으로 벌크 업로드(Import)하는 기능을 MVP에 포함하여 초기 데이터 구축 시간을 단축해야 합니다.
